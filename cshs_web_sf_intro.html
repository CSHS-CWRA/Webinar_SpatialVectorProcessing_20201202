Simple features and their implementation in R

Characteristics of sf objects

  • Simple Features are implemented within R as data frames.

  • Each row represents a feature.

  • Attributes are stored in columns.

  • Geometries are stored as a list-column; each element corresponds to a specific feature.

Reading and writing spatial data

Available drivers

A broad range of drivers is available for reading and writing spatial vector data. Some examples include

  • ESRI shapefile
  • KML
  • netCDF
  • GeoJSON

You can view a full list via the following code:

## Not run:
st_drivers()

st_read()

The main argument in the st_read() function is dsn, the data source name.

Depending on the driver, dsn may be a file name, a folder, or an open database connection.

The function will automatically find the ‘geometry’ type columns for drivers that support it.

st_write()

The main arguments are

  • obj - object of class sf or sfc
  • dsn - name of file/folder/database connection to write data to
  • layer - name of layer to write
  • driver - name of driver to be used (e.g., KML, GeoJSON); if missing, the driver type will be guessed from dsn

Spatial vector data accessible via R packages

Examples include:

  • rnaturalearth - provides access to Natural Earth (http://www.naturalearthdata.com/) public domain vector and raster map data

  • osmdata - provides access to Open Street Maps crowd-sourced geographic data; vector layers are accessible via package

  • ggmap - provides access to base maps from Google Earth, Google Maps, Stamen Maps and OpenStreetMap

  • bcmaps - provides access to a range of vector data sets relevant to British Columbia

sf for GIS operations

Methods available

sf provides methods for a broad range of aspatial, spatial and geometric operations that would typically be performed in stand-alone GIS package. Some examples include:

  • st_crop()
  • st_buffer()
  • st_intersection()
  • st_distance()
  • st_area()

You can view a full list via:

## Not run:
methods(class = "sf")
